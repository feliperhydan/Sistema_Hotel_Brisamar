# Sistema de Administração de Reservas - Hotel Brisamar

## Descrição

O **Sistema de Administração de Reservas do Hotel Brisamar** é uma aplicação desenvolvida para gerenciar as operações básicas de um hotel, incluindo **cadastro de hóspedes, gerenciamento de quartos, reservas e check-in**.  

O objetivo do sistema é **organizar e automatizar o processo de reservas**, garantindo maior eficiência na administração do hotel e melhor controle das informações dos hóspedes.

---

## Funcionalidades

- Cadastro de hóspedes
- Gerenciamento de quartos
- Realização de reservas
- Validação de pagamento de reserva
- Registro de check-in
- Consulta de reservas
- Geração de relatórios administrativos

---

## Tecnologias Utilizadas

- **Linguagem:** Java  
- **Frontend:** JavaScript / React  
- **Backend:** Java  
- **Banco de Dados:** MySQL  
- **Arquitetura:** API REST
- **Desenvolvimento de diagramas:** Lucidchart e Miro

---

## Estrutura do Projeto

O sistema foi modelado utilizando **UML**, com os seguintes diagramas:

- Diagrama de Classes
- Diagramas de Sequência
- Relacionamentos entre entidades do sistema

Principais entidades do sistema:

- Hóspede
- Quarto
- Reserva
- Pagamento
- Check-in
- Recepcionista

---

## Fluxo Básico de Reserva

1. O hóspede solicita uma reserva.
2. O sistema verifica a disponibilidade do quarto.
3. O sistema calcula o valor da reserva.
4. O pagamento é processado.
5. A reserva é registrada no banco de dados.
6. O sistema confirma a reserva ao usuário.

---

## Requisitos do Sistema

- Persistência de dados em banco relacional
- Interface web responsiva
- Validação de dados do usuário
- Conformidade com a LGPD (Lei Geral de Proteção de Dados)
